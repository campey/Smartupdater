<input id="example1" type="text" value="" />

$("#example1").smartupdater({
	url : 'php/getmin.php',
	minTimeout: 5000 // 5 seconds
	}, function (data) {
		$("#example1").val(data);
	}
);
It is a very basic feature. If data isn't changed, Smartupdater doesn't update client side GUI. It simply doesn't call callback function. In this example server returns hours and minutes, so data from server is changed every minute. Because I set poll interval 5 sec. server returns the same data 11 times. But Smartupdater ignors the data and doesn't update view. You can clear input field or type something in it to see that date is refreshed 1 time in a minute
