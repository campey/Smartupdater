This feature let you to save traffic. It allows the request to be successful only if the response has changed since the last request. This is done by checking the Last-Modified header. This technique also checks the 'etag' specified by the server to catch unmodified data.

Those who are familiar with HTTP caching and already looked into the smartupdater source probable would like to ask me a couple of questions:


Q. jQuery AJAX already has "ifModified" option to activate http cache feature. So, why do you re-invent the wheel?

A. There is a bug in jQuery AJAX engine ( http://bugs.jquery.com/ticket/8095 ) and Smartupdater simply can't use this option.

Q. You are talking about "304 Not Modified" http header, right? But neither your code example, nor your live demo shows any "304 Not Modified" http headers. LOL :)

A. Yes, it's funny :) But there is another jQuery AJAX bug (or maybe not bug, but problem which is known as "Opera issue"). Briefly, the problem is: Opera's xhr object returns "304 Not Modified" header in non-standard format, so that jQuery AJAX engine can't interpret it correctly. That's why to avoid using "304" header, I have to implement caching algotrithm directly in Smartupdater.


Please pay attention to the PHP code: Server doesn't send "304 Not Modified" header, but includes ETag header in each reply. Smartupdater gets this ETag, compares with previous one and if they are equal, process responce as if it gets "304 Not Modified" header.

Two http headers are used by Smartupdater to implement this feature: ( ETag / If-None-Match) and ( Last-Modified / If-Modified-Since). They are processed identically, but ETag is more sutable for cache validation (check if content is the same), while Last-Modified more sutable for checking cache freshness ( check content time or revision number).

Server sends headers ETag or Last-Modified and Smartupdater returns the data in If-None-Match or If-Modified-Since headers.

Open debugger and check network activity. You see, that client sends requests every 5 seconds, but server responses with actual data only onece in a minute. The rest server's responses have header and empty message body.
